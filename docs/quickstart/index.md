# Quickstart

Quickstart is the shortest safe path into the ecosystem.

The goal is not to impress you with knobs.
The goal is to get you to a bootable, testable Yggdrasil system without mystery.

You are going to do five things:

1. verify the build machine is ready,
2. write config with `yggcli`,
3. build both server and KDE artifacts,
4. run the smoke bench,
5. boot and inspect the real machine.

Read these in order:

1. prerequisites
2. configure
3. build
4. smoke test
5. first boot

If anything feels vague while reading quickstart, stop and jump to the corresponding wiki page.
The system should feel demanding, not obscure.
