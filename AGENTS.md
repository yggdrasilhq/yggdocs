# AGENTS.md

## Mission

Maintain `yggdocs` as the canonical long-form documentation set for the Yggdrasil ecosystem.

## Boundaries

- Product behavior belongs in the product repos.
- `yggdocs` should explain workflows, architecture, and operator runbooks once they are stable enough to document.
- Keep private infrastructure values out of tracked docs.

## README And Release Notes Contract

- `README.md` should explain what documentation lives here, how to preview it locally, and how it relates to the rest of the ecosystem.
- Favor operator-facing examples, diagrams, and runbooks over generic marketing text.
- Release notes should summarize meaningful docs IA changes, new runbooks, and documentation migrations that affect readers.
- Release pages should reuse curated notes rather than autogenerated text.

## Release Notes Automation

- Keep `CHANGELOG.md` current with `## Unreleased` at the top while work is in flight.
- When cutting a release, move the user-visible notes into an exact version section before or during the tag workstream.
- Release automation should prefer the exact version section and fall back to `Unreleased` so curated notes still publish when the rename is late.
- Release pages should use curated changelog text rather than autogenerated notes.
- Prefer release notes that explain documentation IA or runbook changes readers need to re-learn.
