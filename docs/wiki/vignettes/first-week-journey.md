# Vignette: The First Week With Yggdrasil

Day 1, you boot and everything feels unfamiliar.
By Day 7, you stop thinking in terms of one machine and start thinking in terms of an ecosystem.

## Day 1: Boot and trust nothing

- Build server and KDE ISOs.
- Run smoke checks.
- Boot, import zpool, verify services.

You are not proving perfection. You are proving the system can recover.

## Day 3: First useful container recipe

Pick one practical service and document it end to end.
The goal is repeatability, not novelty.

## Day 5: Your first migration story

Write what changed, what broke, and what made it stable again.
Those notes become tomorrow's wiki recipe.

## Day 7: You have a pattern

Your docs now contain:

- a quickstart path,
- a wiki recipe,
- and a dev note for why the system is shaped this way.

That pattern is the real product.
